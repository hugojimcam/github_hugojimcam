The coding language I had the most knowledge of was p5.js due to my previous creative coding class, so I wanted to use it for my final project. I had some knowledge required for this project, such as using audio files, mapping keys, creating loops, and creating indexes, among other skills. It would be a good opportunity to push myself and learn more. I linked the resources I used below to fill in the gaps. They range from projects I found through the web and Reddit to a project from last quarter. I started using Spotify and attempted to download the audio files using a converter, but Spotify was down. I almost changed my project entirely, but through some research, I realized I could use YouTube and another converter instead. This process was the most tedious because I had to find videos that were under 4 minutes long and had no movement to keep the file size under 5 MB so they would load into p5.js. On a positive note, this saved me from having to download the images and audio files again, as I could download a video file that contains both. I had changed the songs many times, so the audio and image files I had initially used didn't hold much significance. I spent time learning what I could implement to make the code more efficient. It was too complicated for me, but something I could see myself adding in the future. It was easy to get sidetracked and start something but run into a wall and have to turn back. To compensate for my initial idea of using keys to overlap the songs, I created a visualizer with assistance from other projects I found and the p5.js libraries. I added a doodle I made in Procreate as the background image for the title page, ensured all the formatting looked okay, and then finished my project.
